네이밍 규칙

BEM은 Block, Element, Modifer의 줄임말로 일종의 네이밍 컨벤션 입니다.
일반적으로 BEM 규칙을 사용하는 경우가 많습니다.
하지만, BEM 규칙은 네이밍 글자가 길어진다는 단점이 있습니다.
따라서 저희는 컴포넌트를 제외한 요소는 모두 언더바(_)를 이용하고자 합니다.
컴포넌트
독립되어 분리, 재사용 할수 있는 것으로 Block 요소는 물론,
지속적으로 재사용 될 영역과 컨텐츠 모두 컴포넌트로 인식해주시면 됩니다.
ex) header, form, plugin tab, plugin popup, 유의사항, 풀서비스 … 컴포넌트는 기본적으로 하이픈(-)을 사용합니다.

    

        
컨텐츠
컨텐츠 내 사용되는 네이밍
  1. 시멘틱 구조에 의거하여 컨텐츠 내 Block형태는 (el)__wrap을 사용합니다.
    직관적이게 보이기 위해 해당 영역은 언더바(_) 2개를 사용합니다.
    다만, 컴포넌트에서 wrap을 사용할 경우 하이픈(-)을 사용합니다.
    ex) .sub01__wrap , .sub02__wrap …, 컴포넌트 .tab-wrap, .notice-wrap…
  2. Block 안에 Element요소가 여럿 나뉘어져 있을 경우에는 wrap 대신 (el)_box, (el)-box를 사용합니다.
  3. layout을 잡을땐 .inner를 사용합니다.
  4. 많이 쓰이는 공통 용어는 축약어를 사용합니다.
    축약어 관련 사이트
  5. 수평 구조를 이용할 때는 css에서 flex를 위주로 이용합니다.
    float를 이용시에는 공통 클래스를 사용합니다.
    (css/common/common.css에서 확인 가능)
